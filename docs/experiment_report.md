# 金融影像相似度检测实验结论报告

## 1. 实验目标

本实验面向金融影像资料中的面签照片，构建一条可复现的相似度检测流程：先从多类金融影像中筛出面签照片，再用图像向量进行 Top-K 相似检索，最后通过人工审核校准风险阈值。

## 2. 数据规模

本次实验使用仓库内赛题数据集，合计 370 张图片，坏图数量为 0。数据类别分布均衡：

| 类别 | 数量 |
| --- | ---: |
| bank_statement | 74 |
| contract | 74 |
| face_signing | 74 |
| id_card_back | 74 |
| id_card_front | 74 |

流水线最终筛出 74 张面签照片，并仅对这些面签照片建立 FAISS 相似检索索引。

## 3. 模型方法

本项目采用 `google/siglip2-base-patch16-224` 作为图像编码模型，整体流程如下：

1. 扫描数据集并校验图片可读性。
2. 使用 SigLIP2 提取全部有效图片的 embedding。
3. 在 embedding 上训练轻量线性分类头，完成五类金融影像分类。
4. 筛选预测为 `face_signing` 的面签照片。
5. 对面签照片 embedding 使用 FAISS `IndexFlatIP` 建立向量索引。
6. 输出每张面签照片的 Top-K 相似候选及余弦相似度。
7. 基于人工审核结果校准风险阈值。

## 4. 阈值选择

流水线中的代理阈值实验曾给出 `0.98` 作为最佳 F1 阈值，但该实验使用的是“原图轻微裁剪/亮度增强”作为正样本代理，不能完全替代真实人工审核。

随后对候选相似对进行人工复核：

| 区间 | 审核结论 |
| --- | --- |
| `>= 0.98` | 8 组唯一高风险候选全部确认相似 |
| `0.97 ~ 0.98` | 25 组候选全部确认相似 |
| `0.95 ~ 0.97` | 开始出现不稳定候选，不适合作为高风险阈值 |

因此最终采用：

| 风险等级 | 阈值 |
| --- | --- |
| high | `cosine_similarity >= 0.97` |
| medium | `0.93 <= cosine_similarity < 0.97` |
| low | `cosine_similarity < 0.93` |

`0.97` 是人工审核校准后的高风险阈值，不是默认经验值。

## 5. 人工审核结果

当前人工审核标注保存在 `outputs/mvp/review_labels.csv`。

| 审核标签 | 数量 |
| --- | ---: |
| 确认相似 | 33 |
| 确认不相似/不采用 | 11 |
| 合计 | 44 |

在已审核样本中，`>= 0.97` 的 33 组唯一候选均被确认相似；`0.95 ~ 0.97` 区间开始出现误报，因此最终 high 阈值固定为 `0.97`。

## 6. 最终风险分层结果

按最终阈值对 Top-K 检索结果去重统计：

| 风险等级 | 唯一候选对数量 |
| --- | ---: |
| high | 33 |
| medium | 66 |
| low | 155 |

建议业务使用方式：

- high：必须进入人工复核，当前审核样本显示该区间可信度高。
- medium：作为抽检或二审池，适合持续补充人工标签。
- low：默认低风险，仅在业务需要扩大召回时抽样检查。

## 7. 局限与后续工作

当前人工审核样本主要集中在高分区间，已经足以支撑 high 阈值选择，但 medium/low 区间仍可继续抽样补充负样本标签。若后续获得官方相似对标注或更大规模人工标注，可重新计算 Precision、Recall、F1 并进一步校准阈值。
