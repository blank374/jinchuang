# 基于 SigLIP2 与分层风控策略的金融影像智能相似度检测系统

> 赛题：第五届中国研究生金融科技创新大赛相关金融影像相似度检测任务  
> 主模型：`google/siglip2-base-patch16-224`  
> Baseline：`openai/clip-vit-base-patch32`  
> 核心思想：模型判断“像不像”，业务策略判断“危险不危险”

## 1. 方法概述

金融影像相似度检测不能只停留在“图片相似度分数”层面。两张面签照片相似，在不同业务关系下含义并不相同：同一客户续贷时出现相似影像可能是正常复用，而不同客户之间出现相似面签照片则更可能涉及冒用、套用或重复欺诈。

因此，本系统采用两层设计：

1. **视觉语义层**：使用 SigLIP2 提取金融影像语义向量，完成影像类型识别与面签照片相似检索。
2. **业务风控层**：结合客户/业务关系进行分层阈值判断，输出风险等级、复核优先级与处置建议。

整体流程如下：

```text
输入金融影像
-> 预处理增强
-> SigLIP2 影像类型识别
-> 面签照片筛选
-> SigLIP2 图像 embedding
-> L2 归一化
-> FAISS Top-K 检索
-> 同客户/跨客户关系判断
-> 分层阈值风险评估
-> 风险等级 + 复核优先级 + 处置建议
```

## 2. 模型设计

### 2.1 主模型与 baseline

系统当前使用 **SigLIP2** 作为主模型：

| 项目 | 配置 |
|---|---|
| 主模型 | `google/siglip2-base-patch16-224` |
| baseline | `openai/clip-vit-base-patch32` |
| 主模型 embedding 维度 | 768 |
| baseline embedding 维度 | 512 |
| 检索方式 | FAISS 内积检索 |

选择 SigLIP2 作为主模型的原因：

- 相比传统 CLIP，SigLIP2 是更新的视觉语言编码器；
- 对图文语义对齐、细粒度视觉表征和多语言场景更友好；
- 适合作为金融影像分类与相似度检索的统一特征底座。

CLIP 保留为 baseline，用于说明系统从经典多模态模型到主模型增强的演进路径。

### 2.2 影像类型识别

系统定义五类金融影像：

| 类别 ID | 类别 |
|---|---|
| `sign_photo` | 面签照片 |
| `id_card` | 身份证 |
| `contract` | 合同 |
| `bank_statement` | 银行流水 |
| `other` | 其他 |

分类器通过各类别 prompt 生成文本特征，再与输入图像特征计算相似度，选择得分最高的类别。该设计避免了大量标注数据依赖，也便于后续加入新类别。

下游相似度检测只针对 `sign_photo` 进行。身份证、合同、流水等非目标影像会被直接跳过，以减少无效检索和误报。

### 2.3 图像特征与 FAISS 检索

系统使用 SigLIP2 的 image encoder 提取 768 维图像特征，并做 L2 归一化：

```python
features = model.get_image_features(pixel_values=images)
features = normalize(features)
```

归一化后，余弦相似度可以用向量内积表示，因此 FAISS 使用 `IndexFlatIP` 即可完成快速 Top-K 检索。对于更大规模数据，可切换到 IVF 索引。

## 3. 分层风控策略

### 3.1 为什么不能只用统一阈值

统一阈值只能回答：

```text
这两张图像像不像？
```

但金融风控更关心：

```text
这两张图像在业务语境中是否危险？
```

例如：

- 同一客户多次贷款，面签照片相似可能是续贷或复用；
- 不同客户之间面签照片相似，更可能是冒用或套用；
- 相似度处于中间区域时，不宜自动拦截，应进入人工复核。

因此系统引入业务语义感知的分层阈值策略。

### 3.2 当前阈值配置

配置位于 [config.yaml](config.yaml)：

```yaml
retrieval:
  similarity_threshold: 0.94
  dynamic_threshold:
    enabled: true
    fraud: 0.75
    same_customer: 0.92
```

含义如下：

| 场景 | 关系 | 阈值 | 设计目标 |
|---|---|---:|---|
| 跨客户命中 | `cross_customer` | 0.75 | 提高疑似冒用召回 |
| 同客户命中 | `same_customer` | 0.92 | 减少正常续贷误报 |
| 关闭动态阈值 | default | 0.94 | 统一相似度判断 |

### 3.3 风险输出

系统不仅输出相似度，还输出可解释处置字段：

| 字段 | 含义 |
|---|---|
| `risk_level` | 高 / 中 / 低风险 |
| `risk_type` | 风险类型，如跨客户疑似套用 |
| `review_priority` | 复核优先级 |
| `recommended_action` | 处置建议 |
| `risk_summary` | 批量风险摘要 |

示例：

```json
{
  "similarity": 0.8912,
  "relationship": "cross_customer",
  "threshold_applied": 0.75,
  "risk_level": "medium",
  "risk_type": "cross_customer_suspect",
  "review_priority": "standard",
  "recommended_action": "Send to anti-fraud review; verify identity and loan context before approval."
}
```

这使系统从“相似图片搜索工具”升级为“金融影像风险辅助审核系统”。

## 4. 工程实现

### 4.1 模块划分

| 模块 | 文件 | 功能 |
|---|---|---|
| 模型编码器 | [src/model.py](src/model.py) | 按配置加载 SigLIP2 / CLIP |
| 影像分类 | [src/classifier.py](src/classifier.py) | prompt-based 影像类型识别 |
| 向量检索 | [src/retrieval.py](src/retrieval.py) | FAISS Flat / IVF 检索 |
| 预处理 | [src/preprocessing.py](src/preprocessing.py) | EXIF 旋转、对比度增强、透视矫正 |
| 风控策略 | [src/risk_policy.py](src/risk_policy.py) | 分层阈值、风险等级、处置建议 |
| 数据入库 | [ingest.py](ingest.py) | 扫描影像并构建索引 |
| Web 展示 | [main.py](main.py) | Gradio 交互界面 |
| API 服务 | [api.py](api.py) | FastAPI HTTP 接口 |

### 4.2 关键实现点

1. **模型配置化**  
   `config.yaml` 决定实际加载的主模型。代码中保留 `CLIPFeatureExtractor` 类名是为了兼容旧调用，但内部已改为 `AutoModel + AutoProcessor`，可以加载 SigLIP2。

2. **索引维度一致性**  
   SigLIP2 输出 768 维向量，因此 FAISS 索引必须按 768 维重建。旧 CLIP 512 维索引不能直接复用。

3. **同一策略复用**  
   `src/risk_policy.py` 同时供 API 和 Web 页面调用，避免单图检测、批量检测、接口检测出现不同判断口径。

## 6. 实验与评估说明

当前融合分支的代码已切换为 SigLIP2 主模型。因此，旧文档中基于 CLIP 512 维向量得到的指标不能直接视为当前最终指标。重新建索引并跑评估后，应以 SigLIP2 版本的结果更新实验表格。

建议保留以下评估维度：

1. **分类准确率**：五类金融影像识别准确率；
2. **Top-K 命中率**：相似组样本是否能在前 K 个结果中召回；
3. **阈值扫描**：统一阈值下 Precision / Recall / F1；
4. **分层风控效果**：跨客户召回率、同客户误报率、人工复核量；
5. **消融实验**：
   - CLIP baseline vs SigLIP2 主模型；
   - 统一阈值 vs 分层阈值；
   - 无预处理 vs 预处理增强；
   - Flat vs IVF 索引。

## 7. 结论

本系统的核心价值不只是“找到相似图片”，而是将多模态视觉模型能力接入金融业务风控逻辑：

- SigLIP2 负责提供更强的金融影像语义表征；
- FAISS 负责高效召回相似面签影像；
- 分层风控策略负责把相似度转化为业务风险；
- API 和 Web 页面负责给出可解释、可复核、可落地的处置结果。

一句话概括：

> 模型判断影像“像不像”，风控策略判断这件事“危险不危险”。
