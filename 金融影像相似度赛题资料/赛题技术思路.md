# 基于多模态大模型的金融影像智能相似度检测模型 - 技术思路

整理日期：2026-07-06

## 1. 赛题核心

本题不是单纯做图片相似度，而是要做一套可运行的金融影像风控检测流程：

1. 从混合影像中自动识别并筛选“客户经理面签照片”。
2. 将面签照片与历史面签照片比对，发现重复提交、跨客户套用、疑似伪造复用等可疑交易。
3. 输出分类结果、相似度分数、可疑交易列表和检测汇总视图。

推荐整体流程：

```text
原始金融影像
  -> 数据清洗与样本盘点
  -> 多模态影像分类（SigLIP2/CLIP + 分类头）
  -> 自动筛选客户经理面签照片
  -> 深度度量学习特征编码（Triplet Loss / Contrastive Learning）
  -> 向量 Embedding
  -> FAISS Top-K 相似检索
  -> Cosine Similarity 计算
  -> 业务规则融合
  -> 风险评分
  -> 可疑交易输出与可视化
```

该流程中，Cosine Similarity 仅作为归一化向量之间的最终距离计算方式；真正学习判别性 Embedding 的是由视觉编码器与深度度量学习目标共同构成的特征编码网络。FAISS 负责在历史影像库中高效召回 Top-K 候选，业务规则与风险评分模块负责将图像相似性转化为可解释的金融风控结论。

## 2. 数据下来后的第一天工作

优先做数据盘点，不要一上来就训练。

需要确认：

- 图片路径、影像类型标签、相似度标签、业务类型标签是否齐全。
- 是否有交易号、客户号、贷款产品、客户经理、机构、日期等业务字段。
- 每类图片数量，尤其是面签照片占比。
- 正样本、负样本比例，即已知相似/重复图片对数量。
- 是否存在重复文件、损坏图片、超大图片、极小图片、方向旋转、遮挡、水印。
- 标签是否明显有噪声，比如合同图被标成面签照片。

建议先产出：

- `data_profile.csv`：数据规模统计。
- `samples_by_class/`：每个类别抽样图片。
- `bad_images.csv`：打不开或尺寸异常的图片。
- `duplicate_hash.csv`：完全重复文件或感知哈希高度相近图片。

## 3. 数据划分原则

避免数据泄漏很重要。不要简单随机按图片划分。

优先级：

1. 按客户 ID 划分。
2. 如果没有客户 ID，按交易 ID / 业务流水号划分。
3. 如果都没有，再按图片对关系做 group split，保证同一组相似图片不要同时泄漏到训练和测试。

推荐比例：

```text
训练集 70%
验证集 15%
测试集 15%
```

如果正样本很少，可以保留一份固定测试集，训练时只在训练/验证集调参。

## 4. 基线方案

先做一个可跑通的强基线：

```text
CLIP
  -> 提取 image embedding
  -> 影像类型分类器：Logistic Regression / MLP
  -> 面签照片筛选
  -> FAISS Top-K 检索
  -> Cosine Similarity
  -> 阈值判定
```

CLIP 作为 Baseline，主要用于验证预训练视觉语言模型在金融影像分类和相似度检索任务上的零样本或少样本迁移能力。基线阶段冻结视觉编码器，仅训练轻量分类头，并直接使用归一化预训练 Embedding 建立检索结果，为后续 SigLIP2 主模型和度量学习改进提供统一参照。

分类任务：

- 输入：全部影像。
- 输出：面签照片、身份证明、权属证明、合同文档、其他。
- 初版：冻结视觉编码器，只训练轻量分类头。
- 增强：对金融影像数据做少量微调。

相似度任务：

- 输入：已筛选出的面签照片。
- 输出：相似图片对、相似度分数、关联业务信息。
- 初版：归一化 embedding 后计算余弦相似度。
- 工程化：用 FAISS 建向量索引，支持大规模历史库检索。
- 定位：基线中的 Cosine Similarity 不承担特征学习，仅用于度量已有 Embedding 的相近程度。

## 5. 进阶方案

如果数据质量和时间允许，可以做三类增强。

### 5.1 主模型与对比模型

模型角色按以下层次组织：

- **主方案 SigLIP2**：作为核心视觉编码器，连接影像分类头与度量学习投影头。其多语言视觉语言表征能力适合金融影像分类、中文业务语义对齐和图文检索。
- **Baseline CLIP**：作为可快速复现的基线模型，用于衡量主方案相对通用视觉语言预训练模型的提升。
- **对比实验 DINOv3**：作为纯视觉表征模型，重点比较其在局部结构、场景近重复和细粒度视觉变化上的能力。
- **扩展讨论 Chinese-CLIP**：可用于中文类别提示词和业务文本对齐实验，但不作为主方案，避免不必要的模型堆叠。
- **扩展讨论 DINOv2**：可作为 DINOv3 的前代视觉表征参照，在算力或模型可用性受限时用于局部结构和细粒度变化实验。

主方案网络结构：

```text
金融影像
  -> SigLIP2 视觉编码器
  -> 分类头：预测影像类别
  -> 度量学习投影头：学习相似度 Embedding
```

原有多模型融合思路保留为扩展实验，可采用 `concat(CLIP_embedding, DINO_embedding)` 进行特征级融合，或采用 `score = alpha * clip_score + beta * dino_score + gamma * hash_score` 进行分数级融合，但不纳入主流程，避免增加不必要的部署复杂度。

### 5.2 深度度量学习模块

该模块是相似度检测的核心，而 Cosine Similarity 只是其输出向量的距离计算方法。利用已知相似图片对、重复提交记录和人工标注样本训练 Embedding，使同一面签照片及其裁剪、压缩、旋转等变体距离更近，使不同客户、不同交易的面签照片距离更远。

主要训练目标：

- **Triplet Loss**：以 Anchor、Positive、Negative 三元组约束类内距离与类间距离，作为主度量学习损失。
- **Contrastive Loss**：基于相似/不相似图片对直接优化向量距离，适合已有成对标签的数据。
- **ArcFace / CosFace（可选）**：通过角度间隔增强身份或场景级特征的判别边界；在标签粒度和样本量允许时使用。
- **Supervised Contrastive Learning（扩展）**：在一个批次内利用多个正负样本，提高特征空间的聚类质量和训练稳定性。

目标：

```text
相同或复用的面签照片距离更近
不同交易、不同人员、不同场景距离更远
```

Hard Negative 选择是训练重点，优先选择：

- 同一网点但人员不同的面签照片。
- 同一客户经理参与但客户不同的面签照片。
- 背景、桌面、拍摄角度相似但人员不同的照片。
- FAISS 初步检索中相似度较高、经人工确认不相似的误报样本。

注意事项：

- 训练时可联合优化分类损失与度量学习损失：`Loss = lambda_cls * L_cls + lambda_metric * L_metric`。
- 训练集必须避免把测试集相似对泄漏进来。

### 5.3 近重复检测组合策略

实际业务里“重复使用”可能有裁剪、压缩、亮度变化、轻微旋转、截图二传。

建议组合：

```text
感知哈希 pHash / dHash：抓完全重复和轻微压缩
深度 embedding：抓语义相似和画面结构相似
人脸/人体区域特征：辅助判断同一面签场景
OCR/水印/时间戳：辅助发现同图改业务信息
```

比赛阶段不一定要全做，但报告里可以作为扩展设计。

#### 5.3.1 业务特征融合

模型不仅使用图像 Embedding，还融合贷款产品、业务类型、机构、客户经理、交易日期等结构化业务字段，以体现真正的多模态金融风控能力。

业务相似度可通过以下方式获得：

- 对低基数类别字段进行 One-Hot 编码。
- 对业务标签和结构化字段学习 Business Embedding。
- 将业务描述、产品名称等文本输入文本编码器获得语义向量。
- 对交易日期构造时间间隔、是否同日或是否处于异常短周期等规则特征。

最终风险评分采用可解释的加权融合策略：

```text
FinalScore = alpha * ImageSimilarity + beta * BusinessSimilarity
```

其中，`ImageSimilarity` 来自度量学习 Embedding 的相似度，`BusinessSimilarity` 来自业务字段编码或业务规则。实际实现中可在验证集上学习权重，并对同机构、同客户经理、短时间跨客户复用等高风险组合增加规则修正项。

#### 5.3.2 OCR 辅助模块（增强方案）

对于身份证、合同、权属证明等文档类图片，可增加 OCR 文本识别模块。OCR 结果可用于辅助影像分类、辅助文档相似度判断，以及识别“底图相同但姓名、证件号、金额或日期被修改”的同图改字行为。

OCR 作为比赛阶段的增强模块，与面签照片主流程解耦，不影响主方案按期交付。其输出可通过文本 Embedding、关键词匹配或字段一致性规则并入业务相似度。

#### 5.3.3 人脸区域辅助特征（可选增强）

对于面签照片，可在整体场景特征之外增加 Face ROI 特征：

```text
Face Detection
  -> Face ROI
  -> Face Embedding
  -> 与整体 Scene Embedding 融合
```

最终相似度可表示为：

```text
Score = alpha * SceneSimilarity + beta * FaceSimilarity
```

Scene Embedding 用于识别相同拍摄环境、构图和影像复用，Face Embedding 用于辅助区分背景相似但人员不同的 Hard Negative。该能力属于可选增强，不作为完成赛题主流程的前置条件。

## 6. 阈值策略

赛题明确要求说明相似度阈值，因此必须做阈值实验。

建议输出：

```text
threshold, precision, recall, f1, false_positive_rate, false_negative_rate
0.75
0.80
0.85
0.90
0.95
```

结论写法：

- 阈值低：召回率高，能抓更多可疑交易，但误报增加。
- 阈值高：准确率高，人工复核压力低，但可能漏检。
- 金融风控场景通常更重视召回，可选择较低阈值进入人工复核队列，再按高相似度排序。

推荐业务策略：

```text
score >= 0.95：高度可疑，优先审核
0.90 <= score < 0.95：中度可疑，进入复核池
0.85 <= score < 0.90：低度可疑，结合业务规则判断
score < 0.85：默认正常
```

具体阈值要用验证集 PR 曲线、ROC 曲线和业务复核成本确定。

阈值选择应综合验证集 PR 曲线、ROC 曲线、F1 值、误报/漏报成本以及人工审核容量。金融风控场景通常更关注 Recall，因此采用“双阈值 + 分层处置”策略：较低阈值保障高召回并进入人工复核，高阈值样本直接标记为高风险并优先审核。所有阈值均需在独立验证集确定，在测试集上只做一次最终评估。

## 8. 系统交付形态

建议做一个轻量演示系统：

```text
后端：FastAPI
模型：PyTorch / transformers / open_clip / faiss
前端：Vue / React / Streamlit 均可，比赛时间紧可用 Streamlit
存储：SQLite / CSV / parquet
```

页面功能：

- 数据概览：影像总数、面签照片数、疑似风险交易数。
- 分类结果：每张图的预测类别和置信度。
- 可疑交易列表：交易号、客户号、业务类型、相似度、风险等级。
- 图片对比：当前图片、历史相似图片、相似度分数、关联业务字段。
- Top-K 检索：查看查询影像及其历史库中最相似的 K 个候选。
- 风险等级：展示低、中、高风险分层及对应处置状态。
- 阈值调节：动态展示不同阈值下可疑数量、Precision、Recall 和预计人工复核量的变化。
- 可疑交易详情：展示交易时间、机构、客户经理、贷款产品、命中规则及风险评分构成。

Dashboard 形成从数据统计、分类结果、相似图片展示、Top-K 检索、风险分级、阈值调节到可疑交易详情的完整闭环。系统应同时保留模型分数、业务规则命中信息和人工复核结论，满足结果可解释、过程可追溯和后续迭代需要。

## 11. 可做的消融实验

- CLIP vs SigLIP2 vs DINOv3，分别对应 Baseline、主模型和视觉表征对比实验。
- 不加入 Metric Learning vs 加入 Triplet Loss / Contrastive Loss。
- 仅图像相似度 vs 加入 Business Fusion。
- 只用分类筛选 vs 分类筛选 + 相似度检测。
- 只用深度特征 vs 深度特征 + 感知哈希。
- 冻结编码器 vs 微调编码器。
- 不同阈值对 Precision、Recall、F1 和人工复核量的影响。

实验应采用统一数据划分、相同候选库和一致评价指标，每次只改变一个变量，以量化度量学习、业务融合、Hash 辅助、阈值策略和编码器微调的独立贡献。

## 12. 当前推荐路线

比赛实现优先级：

```text
第一优先级：CLIP Baseline + SigLIP2 分类器 + 面签照片自动筛选
第二优先级：深度度量学习 + FAISS Top-K + Cosine Similarity + 阈值实验
第三优先级：业务特征融合 + 风险评分 + Dashboard 可视化
扩展优先级：DINOv3 对比实验 + 感知哈希 + OCR + Face ROI 辅助
```

最稳妥的技术主线：

```text
以 SigLIP2 视觉编码器为核心完成金融影像类型识别，
自动筛选面签照片后通过深度度量学习获得判别性向量表示，
使用 FAISS Top-K 检索和 Cosine Similarity 发现历史复用，
再融合业务特征与规则形成风险评分，
通过阈值分层和 Dashboard 可视化支撑人工复核。
```

综上，本方案以 SigLIP2 视觉编码器为核心，采用“影像分类 -> 面签筛选 -> 深度度量学习 -> 向量检索 -> 业务融合 -> 风险评分”的技术路线。方案以 CLIP 为 Baseline、DINOv3 为对比实验，避免无效的模型堆叠，同时保留 OCR、Face ROI 和感知哈希等可插拔增强能力。整体设计兼顾分类准确率、相似度检测精度、风险结果可解释性和工程部署效率，符合金融影像风控实际需求，并覆盖赛题关于影像分类、相似度检测、阈值分析、系统实现和消融实验的要求。
