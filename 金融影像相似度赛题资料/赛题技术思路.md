# 基于多模态大模型的金融影像智能相似度检测模型 - 技术思路

整理日期：2026-07-06

## 1. 赛题核心

本题不是单纯做图片相似度，而是要做一套可运行的金融影像风控检测流程：

1. 从混合影像中自动识别并筛选“客户经理面签照片”。
2. 将面签照片与历史面签照片比对，发现重复提交、跨客户套用、疑似伪造复用等可疑交易。
3. 输出分类结果、相似度分数、可疑交易列表和检测汇总视图。

推荐整体流程：

```text
原始影像
  -> 数据清洗与样本盘点
  -> 影像类型分类
  -> 筛选面签照片
  -> 图像向量编码
  -> 向量库检索 / TopK 相似图片
  -> 阈值判定可疑样本
  -> 结果汇总与可视化展示
```

## 2. 数据下来后的第一天工作

优先做数据盘点，不要一上来就训练。

需要确认：

- 图片路径、影像类型标签、相似度标签、业务类型标签是否齐全。
- 是否有交易号、客户号、贷款产品、客户经理、机构、日期等业务字段。
- 每类图片数量，尤其是面签照片占比。
- 正样本、负样本比例，即已知相似/重复图片对数量。
- 是否存在重复文件、损坏图片、超大图片、极小图片、方向旋转、遮挡、水印。
- 标签是否明显有噪声，比如合同图被标成面签照片。

建议先产出：

- `data_profile.csv`：数据规模统计。
- `samples_by_class/`：每个类别抽样图片。
- `bad_images.csv`：打不开或尺寸异常的图片。
- `duplicate_hash.csv`：完全重复文件或感知哈希高度相近图片。

## 3. 数据划分原则

避免数据泄漏很重要。不要简单随机按图片划分。

优先级：

1. 按客户 ID 划分。
2. 如果没有客户 ID，按交易 ID / 业务流水号划分。
3. 如果都没有，再按图片对关系做 group split，保证同一组相似图片不要同时泄漏到训练和测试。

推荐比例：

```text
训练集 70%
验证集 15%
测试集 15%
```

如果正样本很少，可以保留一份固定测试集，训练时只在训练/验证集调参。

## 4. 基线方案

先做一个可跑通的强基线：

```text
CLIP / Chinese-CLIP / SigLIP2 / DINOv3
  -> 提取 image embedding
  -> 影像类型分类器：Logistic Regression / MLP
  -> 面签照片筛选
  -> cosine similarity
  -> TopK 检索
  -> 阈值判定
```

分类任务：

- 输入：全部影像。
- 输出：面签照片、身份证明、权属证明、合同文档、其他。
- 初版：冻结视觉编码器，只训练轻量分类头。
- 增强：对金融影像数据做少量微调。

相似度任务：

- 输入：已筛选出的面签照片。
- 输出：相似图片对、相似度分数、关联业务信息。
- 初版：归一化 embedding 后计算余弦相似度。
- 工程化：用 FAISS 建向量索引，支持大规模历史库检索。

## 5. 进阶方案

如果数据质量和时间允许，可以做三类增强。

### 5.1 多模型特征融合

尝试：

- CLIP / Chinese-CLIP：适合用文本提示做零样本分类。
- SigLIP2：多语言视觉语言编码器，适合中文业务语义和图文检索。
- DINOv2 / DINOv3：视觉表征强，适合图像相似、局部结构、细粒度变化。

融合方式：

```text
embedding = concat(CLIP_embedding, DINO_embedding)
或
score = alpha * clip_score + beta * dino_score + gamma * hash_score
```

### 5.2 度量学习微调

如果数据提供了相似/不相似图片对，可做：

- Contrastive Loss
- Triplet Loss
- ArcFace / CosFace
- Supervised Contrastive Learning

目标：

```text
相似/重复面签照片距离更近
不同交易、不同人员、不同场景距离更远
```

注意事项：

- Hard negative 很关键，比如同一网点、同一客户经理、类似背景但不同客户的照片。
- 训练集必须避免把测试集相似对泄漏进来。

### 5.3 近重复检测组合策略

实际业务里“重复使用”可能有裁剪、压缩、亮度变化、轻微旋转、截图二传。

建议组合：

```text
感知哈希 pHash / dHash：抓完全重复和轻微压缩
深度 embedding：抓语义相似和画面结构相似
人脸/人体区域特征：辅助判断同一面签场景
OCR/水印/时间戳：辅助发现同图改业务信息
```

比赛阶段不一定要全做，但报告里可以作为扩展设计。

## 6. 阈值策略

赛题明确要求说明相似度阈值，因此必须做阈值实验。

建议输出：

```text
threshold, precision, recall, f1, false_positive_rate, false_negative_rate
0.75
0.80
0.85
0.90
0.95
```

结论写法：

- 阈值低：召回率高，能抓更多可疑交易，但误报增加。
- 阈值高：准确率高，人工复核压力低，但可能漏检。
- 金融风控场景通常更重视召回，可选择较低阈值进入人工复核队列，再按高相似度排序。

推荐业务策略：

```text
score >= 0.95：高度可疑，优先审核
0.90 <= score < 0.95：中度可疑，进入复核池
0.85 <= score < 0.90：低度可疑，结合业务规则判断
score < 0.85：默认正常
```

具体阈值要用验证集 PR 曲线、ROC 曲线和业务复核成本确定。

## 7. 评价指标

分类任务：

- Accuracy
- Precision
- Recall
- F1-score
- Confusion Matrix

相似度检测：

- AUC
- Precision / Recall / F1
- Precision@K
- Recall@K
- Top-1 / Top-5 命中率
- mAP 或 Recall@K，可用于检索评价

系统性能：

- 单张图片编码耗时
- 批量建库耗时
- TopK 检索耗时
- 每万张图片检测耗时

## 8. 系统交付形态

建议做一个轻量演示系统：

```text
后端：FastAPI
模型：PyTorch / transformers / open_clip / faiss
前端：Vue / React / Streamlit 均可，比赛时间紧可用 Streamlit
存储：SQLite / CSV / parquet
```

页面功能：

- 数据概览：影像总数、面签照片数、疑似风险交易数。
- 分类结果：每张图的预测类别和置信度。
- 可疑交易列表：交易号、客户号、业务类型、相似度、风险等级。
- 图片对比：当前图片、历史相似图片、相似度分数、关联业务字段。
- 阈值调节：不同阈值下可疑数量变化。

## 11. 可做的消融实验

- CLIP vs Chinese-CLIP vs SigLIP2 vs DINOv3。
- 只用分类筛选 vs 分类筛选 + 相似度检测。
- 只用深度特征 vs 深度特征 + 感知哈希。
- 冻结编码器 vs 微调编码器。
- 不同阈值对准确率、召回率和人工复核量的影响。

## 12. 当前推荐路线

比赛实现优先级：

```text
第一优先级：CLIP/SigLIP2/DINOv3 embedding + 分类器 + cosine similarity + FAISS
第二优先级：阈值实验 + 可疑交易可视化
第三优先级：度量学习微调 + 多模型融合 + 感知哈希辅助
```

最稳妥的技术主线：

```text
用多模态/视觉基础模型完成金融影像类型识别，
筛出面签照片后使用深度向量表示和向量检索发现历史复用，
再通过阈值分层和可视化结果支撑人工复核。
```
